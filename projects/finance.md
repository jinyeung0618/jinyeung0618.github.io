## 시작 동기

주식을 처음 시작하면서 *매일 아침 무엇을 봐야 할지* 막막했다. 뉴스를 일일이 찾기엔 시간이 없고, 유료 리포트는 부담스럽다. 동시에 *과거 흐름과 그 시점의 사건*을 한 화면에서 보고 싶었다. 그래서 직접 만들었다.

핵심 원칙:

- **사실 기반** — 검색 결과·공개 데이터만 사용. 추측·예측 단언 금지
- **간결함** — 출근길 5분 통독 가능한 길이
- **자동화** — 매일 사람 손 거치지 않고 발행
- **한 화면에 한 섹션** — 보고 싶은 것만 집중

## 구성

탭으로 분리된 두 섹션:

### 📰 모닝 리포트
매일 평일 오전 10:03, 자동 발행. 오늘의 시장 흐름·watchlist 6종목·핵심 변수를 5분 안에 통독 가능한 형태로 정리.

### 📊 지수·종목 통계
종목별 상장일~현재 가격 흐름을 로그 스케일 라인 차트로. 주요 변곡점(고점·저점)에 마커 + 호버 시 그 시점의 사건이 툴팁으로 뜸. 해외/국내 지역 필터.

## 시스템 구성 (모닝 리포트)

```
            매일 평일 10:03 KST
                 ↓
            cron 트리거
                 ↓
    ┌──── Claude (Claude Code) ────────┐
    │  WebSearch × 10회                │
    │  (거시 뉴스 + 종목별 뉴스)         │
    │           ↓                      │
    │  마크다운 리포트 작성             │
    │  (Jekyll frontmatter 포함)       │
    │           ↓                      │
    │  git add + commit + push         │
    └──────────────────────────────────┘
                 ↓
       GitHub Pages 자동 빌드
                 ↓
        사이트 업데이트 (1-2분)
```

## 통계 차트 구현

- **Chart.js** (CDN, 약 20KB) — 정적 사이트에 적합
- **로그 스케일** Y축 — 41년간 0.07→278 배 차이 시각화
- **변곡점 마커** — 이벤트 dates만 점 표시, 호버 시 툴팁에 헤드라인
- **데이터** — 참조점 → 월별 보간 (~500 포인트). 실데이터 연동은 다음 단계
- **다크 모드** — MutationObserver로 테마 변경 시 차트 자동 재렌더

## 결정 로그

- **출처 URL은 본문에 박지 않음** — 가독성 우선. 요청 시 별도 제공.
- **수치 디테일 최소화** — 매출·EPS·capex 같은 분석가용 숫자 제외. 흐름과 방향성만.
- **추측 금지** — "~할 가능성이 높다" 같은 표현 금지. AI가 가장 빠지기 쉬운 함정.
- **평일만 발행** — 토·일은 시장 휴장. 월요일에 주말 누적 뉴스 정리.
- **6종목 watchlist** — 너무 많으면 리포트가 길어지고 결정 피곤해진다.
- **변곡점만 차트 마커** — 모든 사건이 아니라 *방향이 바뀐 순간*만. 신호 vs 노이즈 분리.
- **한 화면 = 한 섹션** — 두 섹션을 동시에 보여주면 정보 과부하. 탭으로 분리.

## 한계와 다음 단계

**현재 한계**
- 차트 데이터는 데모용 근사치 (실시간 데이터 연동 X)

- 페이월 매체(NYT, WSJ) 본문 접근 불가 → 헤드라인·요약만

**다음 단계 후보**
- 실시간 가격 데이터 연동 (Stooq / Alpha Vantage)
- GitHub Actions로 cron 이전 → 클라우드 실행
- 종목별 일별 줌·팬
- 투자 학습 일지 섹션 추가 (3번째 탭)
